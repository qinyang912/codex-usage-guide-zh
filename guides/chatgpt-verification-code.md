# ChatGPT 验证码、OTP 与 MFA 排查清单

更新日期：2026-07-21。

## 先判断验证码来自哪里

| 页面或提示 | 常见用途 | 优先处理 |
| --- | --- | --- |
| 邮箱 6 位 OTP | 新设备、异常位置或敏感操作的登录验证 | 只使用最新邮件，检查垃圾箱和企业邮箱隔离区 |
| ChatGPT App 登录审批 | 已登录设备确认新登录 | 打开官方 App；没有推送时改用邮箱 OTP |
| MFA 验证码 | 已启用多因素认证的账号 | 选择 `Try another method`，使用已配置的备用方式 |
| 短信或 WhatsApp | 特定手机号验证流程 | 仅使用页面明确提供的方式，不寻找绕过服务 |
| 恢复码 | MFA 或高级账号安全恢复 | 离线保存，只在 OpenAI 官方页面输入 |

## 邮箱验证码收不到

1. 确认仍在使用注册时相同的登录方式，例如 Google、Apple、Microsoft 或邮箱密码。
2. 等待最新邮件送达，只输入最新一封邮件中的验证码。
3. 检查垃圾邮件、推广分类和企业邮箱的安全隔离区。
4. 将 `noreply@tm.openai.com` 和 `otp@tm1.openai.com` 加入允许列表。
5. 如果已经无法访问原邮箱，不要把短信验证当成邮箱登录验证的通用替代方案，应通过 OpenAI Help Center 联系支持。

## MFA 设备丢失或无法登录

在验证页面选择 `Try another method`，查看账号之前启用的验证器、推送、短信 / WhatsApp 或 Passkey。具体选项会随设备、地区、套餐和账号而不同。

如果账号启用了高级账号安全，恢复要求可能更严格。恢复码应事先离线保存；没有可用验证方式和恢复码时，支持团队也未必能按普通流程恢复账号。

## 手机验证要区分 ChatGPT 和 API

OpenAI 当前帮助文档说明，创建和使用普通 ChatGPT 账号通常不再要求手机号；但在 OpenAI Platform 首次生成 API Key 时仍可能要求手机验证。官方支持的方式是短信，部分地区可以使用 WhatsApp，不支持用邮件或语音电话替代。

不要购买临时手机号、验证码接收或伪装地区服务。这些做法可能增加账号丢失、地区不一致和安全审查风险。

## 收到自己没有发起的验证码

- 不要输入、转发或截图发送验证码。
- 对陌生 App 登录审批选择拒绝。
- 立即更新密码，并检查 Active sessions、MFA 与安全设置。
- 如果邮箱本身也有异常登录，先保护邮箱账号。

## 第三方充值边界

邮箱验证码、短信验证码、MFA 验证码和恢复码都属于高敏感认证信息，不应发送给代充卖家。即使卖家声称“只用一次”或“不会保存”，验证码也可能直接批准一次登录。
