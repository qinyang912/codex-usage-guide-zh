# ChatGPT / Codex 账号停用与申诉检查清单

更新日期：2026-07-21。

## 先判断错误类型

| 提示或现象 | 更可能属于 | 优先处理 |
| --- | --- | --- |
| Account Deactivated / 账号已停用 | OpenAI 账号停用 | 查看通知邮件并进入官方申诉 |
| Deleted or Deactivated | 本人删除或平台停用 | 查找删除确认邮件与停用通知 |
| Unsupported Country | 地区支持或误判 | 核对 OpenAI 当前支持国家列表 |
| 验证码或 MFA 失败 | 登录验证 | 按原登录方式和备用验证方式排查 |
| 已扣款仍显示 Free | 账号或订阅渠道 | 核对网页、Apple、Google 与购买账号 |
| Business 工作区无法访问 | 工作区席位或管理员状态 | 联系 Workspace Owner / 管理员 |

## OpenAI 当前列出的潜在停用原因

- 违反使用政策。
- 违反服务条款，包括规避安全或访问限制、欺诈或诈骗。
- 账号出现安全隐患或未授权访问。
- 收到警告后仍重复违规。
- 未完成要求的身份或年龄验证。

具体原因应以账号收到的 OpenAI 通知和审核为准，不应只依据第三方“封号原因”文章猜测。

## 官方申诉顺序

1. 优先使用停用通知邮件中的申诉链接。
2. 无法访问该邮件时，使用 [OpenAI Appeal Form](https://openai.com/form/appeal/)。
3. 没有收到通知邮件但已经无法访问时，通过 [Help Center](https://help.openai.com/) 聊天联系支持。

建议只提交一份事实完整的申诉，后续在同一工单补充材料。不要群发公开邮箱、每天重复创建工单或找第三方代替账号持有人申诉。

## 申诉材料

- 账号邮箱、原登录方式、User ID 与 Organization ID；无法获取时明确说明。
- 使用的是 ChatGPT、Codex App / CLI，还是 OpenAI Platform API。
- 停用通知、准确时间、时区、错误代码和此前的警告。
- 正常使用背景，以及疑似被入侵时的日期范围、未知设备或异常用量。
- 已完成的改密、邮箱保护、会话清理、第三方授权与 API Key 撤销。
- 涉及未知扣费时，按官方要求提供日期、金额、卡片后四位、品牌、发卡行和账单邮编。

不要在 GitHub issue、闲鱼聊天、公开截图或第三方表单中发送完整卡号、CVC、验证码、Session、Cookie、恢复码或 API Key。

## Codex 与 API 分支

使用 ChatGPT 登录 Codex 时，停用可能关联同一个 OpenAI 账号。使用 API Key 时还应检查 Platform 的 Organization、Project、Usage、账单和未知 Key。第三方 API 网关不是官方账号恢复路径，也不能替代申诉。

不要通过购买成品号、共享号或“安全中转”规避审核。这会带来新的账号归属、付款和数据隐私风险。

## 订阅与退款

账号停用不代表 Plus 或 Pro 必然自动退款。无法进入 ChatGPT 也不代表 Apple 或 Google Play 订阅已经停止：

- chatgpt.com 网页订阅：通过 OpenAI Help Center 核对取消和退款。
- Apple App Store：在 Apple ID 订阅管理和 Apple 退款渠道处理。
- Google Play：在 Google Play 管理订阅，并按 OpenAI 当前指引申请退款。
- 第三方订单：在原交易平台按订单约定处理，不另付所谓解封费。
